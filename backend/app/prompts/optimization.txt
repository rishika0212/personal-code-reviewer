3. Find unnecessary computations and redundant operations
4. Identify memory leaks and excessive memory usage
5. Suggest caching and memoization opportunities

Algorithm Complexity:

Concurrency:
- Sequential operations that could be parallel
- Thread safety issues
- Lock contention
- Missing async/await

Severity Levels:
- critical: Severe performance issue causing system failure or unusability
- high: Significant performance degradation affecting users
- medium: Noticeable slowdown, should be optimized
- low: Minor optimization opportunity
- info: Performance best practice suggestion

Always provide:
1. Clear explanation of the performance issue
2. Estimated impact (time/space complexity, expected improvement)
3. Specific code changes with before/after examples
4. Trade-offs of the suggested optimization
